Portfolio OLAP Project

"Design a beautiful, modern, and highly intuitive OLAP-style portfolio analytics tool that allows users to easily drill down into their investment portfolio organized by 'sleeve'. Sleeves include categories such as Commodities, Managed Futures, Trend Following, Large Cap Tech, Large Cap Growth, International, Emerging Markets, and others.

The tool must have a professional, attractive interface and be very easy to use, with seamless drill-down from overall sleeves down to individual holdings.

Data Input: Users will import their portfolio data via CSV files exported from brokers like Schwab or Fidelity. The CSV will contain, at minimum, current shares held, current share price, and asset name or ticker. It may also include purchase date, cost basis, and realized gains.

The CSV may or may not include the sleeve/category for each asset. The application must handle both cases gracefully. If the sleeve is missing, provide two solutions: allow the user to manually assign sleeves through a simple interface, and ideally, have the application automatically determine the most appropriate sleeve for each asset.

Please evaluate and give your best recommendation for two paths:

- For a web-based version: What modern tech stack would you recommend, specifically mentioning whether React is the best choice and what complementary libraries or frameworks you'd use.
- For a desktop version: What framework would you recommend, specifically mentioning whether PyQt or Qt is the best approach and why.

After comparing both options, please clearly recommend which platform you think is better overall for this use case and explain your reasoning. Include how the chosen solution should handle CSV file upload, parsing, and sleeve assignment."
